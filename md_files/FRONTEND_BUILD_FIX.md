# Frontend Docker Build Fixes

This document summarizes the changes made to resolve the `vite build` failure and `pnpm` timeout errors when building the frontend Docker container.

## 1. Created `frontend/.dockerignore`
Added `node_modules` and `dist` to this file. This stops Docker's `COPY . .` command from dragging your host Windows `node_modules` into the container and corrupting the Linux-based `node_modules` that `pnpm` just generated. This was the primary cause of the "Cannot find package 'vite'" error.

## 2. Downgraded `pnpm` inside Docker
Replaced `corepack enable pnpm` (which pulled pnpm v11) with `npm install -g pnpm@8.15.5` in the `frontend/Dockerfile`. This avoids strict supply-chain policy checks that time out and fail the build whenever Docker experiences a slow network fetch.

## 3. Fixed Silent Installation Failures
Removed the `|| true` from the `RUN pnpm install` step. That flag was dangerously masking installation failures and causing the build to crash later with confusing missing package errors.

## 4. Added `ENV CI=true`
Explicitly tells `pnpm` not to expect a human to respond to interactive prompts inside the Docker container.

## 5. Used `--no-frozen-lockfile`
Changed the installation command to `pnpm install --no-frozen-lockfile`. This allowed `pnpm v8` to safely regenerate the lockfile inside the container, avoiding version mismatches with the existing lockfile generated by a newer version of pnpm.
